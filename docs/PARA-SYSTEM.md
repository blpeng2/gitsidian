# Gitsidian PARA 시스템

## 개요

Gitsidian은 PARA 방법론에서 영감을 받은 노트 분류 시스템을 제공합니다.
모든 노트(GitHub repo)는 4개의 카테고리 중 하나에 속합니다.

## 카테고리

### 📥 Inbox

- 모든 새 노트의 기본 카테고리
- 외부에서 가져온 정보, 빠른 메모, 아이디어
- 정기적으로 검토하여 다른 카테고리로 이동

### 📌 Active

- 현재 진행 중인 프로젝트/주제
- 자주 참조하고 수정하는 노트
- 활발하게 작업 중인 내용

### 📚 Reference

- 장기 참조 자료
- 다른 노트에서 자주 링크되는 허브 노트
- 언제든 참고할 수 있는 지식 베이스

### 🗃️ Archive

- 완료된 프로젝트
- 더 이상 활발히 사용하지 않는 노트
- 필요시 다시 활성화 가능

## 기술 구현

### 카테고리 저장 방식

GitHub 토픽(topic)으로 관리됩니다:

- `inbox`, `active`, `reference`, `archive`
- 하나의 노트에는 하나의 카테고리 토픽만 부여
- 카테고리 변경 시 기존 카테고리 토픽을 제거하고 새 토픽 추가
- 카테고리 외 다른 토픽은 유지됨

### 자동 분류 규칙 엔진

`src/utils/categoryRules.ts`에 정의된 규칙:

| 규칙 | 조건 | 추천 카테고리 |
|------|------|---------------|
| 활발한 Inbox 노트 | inbox + 7일 내 수정 + 내용 100자 이상 | → Active |
| 허브 노트 | 백링크 3개 이상 | → Reference |
| 비활성 Active 노트 | active + 14~30일 미수정 | → Reference |
| 오래된 노트 | 30일 이상 미수정 | → Archive |

추천은 💡 아이콘으로 표시되며, 사용자가 수동으로 확인 후 이동합니다.

### 카테고리 이동 방법

1. **사이드바**: 노트에 마우스 올리면 카테고리 이동 버튼 표시
2. **토픽 편집**: 에디터 툴팁의 # 버튼에서 직접 토픽 수정

### AI 보조 분류 (반자동)

macOS 앱의 AI 패널에서 프리셋 프롬프트를 사용:

- "이 노트를 Inbox/Active/Reference/Archive 중 어디에 분류하면 좋을지 추천해줘"
- AI 응답을 확인 후 사용자가 직접 카테고리 이동

## 워크플로우

```
1. 새 노트 생성 → 자동으로 📥 Inbox
2. 내용 작성 → 규칙 엔진이 💡 추천 표시
3. 추천 확인 → 적절한 카테고리로 이동
4. 주기적 리뷰 → Inbox 비우기, Archive 정리
```

## 사이드바 UI

카테고리 탭으로 필터링:

- 📋 All — 모든 노트
- 📥 Inbox — 미분류 노트
- 📌 Active — 진행 중
- 📚 Reference — 참조 자료
- 🗃️ Archive — 보관함

각 탭에 해당 카테고리의 노트 개수가 표시됩니다.

## 파일 구조

| 파일 | 역할 |
|------|------|
| `src/utils/categoryRules.ts` | 규칙 엔진, 카테고리 유틸리티 |
| `src/types/index.ts` | NoteCategory, CategoryRecommendation 타입 |
| `src/components/RepoList.tsx` | 카테고리 탭 + 그룹핑 UI |
| `src/App.tsx` | 카테고리 상태 관리 + 핸들러 |

## Second Brain 활용 팁

1. **Inbox Zero**: 매일 Inbox를 비우는 습관
2. **위키링크**: `[[노트이름]]`으로 노트 간 연결 → Reference 허브 형성
3. **토픽 태깅**: 관련 주제 태그로 그래프 연결 강화
4. **정기 리뷰**: 주 1회 Archive 후보 검토
5. **AI 활용**: 분류가 애매한 노트는 AI에게 상담
